ODCS-FusionTool is a standalone tool for execution of data fusion and conflict resolution on RDF data from single or multiple sources. ODCS-FusionTool is based on the Conflict Resolution component implemented in ODCleanStore, a Linked Data management tool.

The main application entry point are classes ODCSFusionToolApplication and ODCSFusionToolExecutor. Other classes are structured into subpackages as follows

config
Classes holding configuration of ODCS-FusionTool and parsing of XML configuration file.
io
Classes providing access to RDF inputs and outputs (files, SPARQL endpoints etc.).
loaders
Classes executing loading of data from input data sources.
urimapping
Classes implementing construction of canonical URI mapping.
util and exceptions
Utilities and exceptions.

For an in-depth description of individual classes and their interfaces, refer to this javadoc documentation. The configuration file accepted by ODCS-FusionTool is documented in form of comments inside the example configuration files distributed with ODCS-FusionTool.
